## 4.2 프로젝트 회고

### 잘 된 점

* 시간표 등록, 조회, 수정, 삭제 기능을 안정적으로 구현하였다.
* JSON 파일 저장 및 불러오기 기능을 적용하여 데이터 관리가 가능하도록 하였다.
* 입력 예외 처리와 중복 시간 검사를 통해 프로그램의 안정성을 높였다.
* 기능별 클래스를 분리하여 유지보수가 용이하도록 설계하였다.

### 아쉬운 점 / 개선 사항

* 실시간 수업 알림 기능이 완전하게 구현되지 못하였다.
* GUI 환경이 아닌 콘솔 프로그램으로 개발되어 사용자 편의성이 부족하다.
* 모바일 알림 및 캘린더 연동 기능을 추가하지 못하였다.

### 향후 개선 계획

* Java Swing을 활용한 GUI 버전 개발
* 실시간 알림 기능 개선
* 캘린더 연동 기능 추가
* 데이터베이스 연동을 통한 데이터 관리 기능 확장
