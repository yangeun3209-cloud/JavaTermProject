# 7. Release

## 1. Release 개요

GitHub Release는 프로젝트의 특정 버전을 배포하기 위한 기능이다.

Release를 사용하면 완성된 프로젝트 버전을 태그(Tag)와 함께 관리할 수 있으며, 사용자는 원하는 버전을 쉽게 다운로드할 수 있다.

---

## 2. Release 생성 목적

시간표 알림 프로그램의 최종 구현 버전을 배포하기 위해 Release를 생성하였다.

Release를 통해 다음 내용을 관리할 수 있다.

* 프로젝트 버전 관리
* 최종 소스 코드 배포
* 기능 변경 사항 기록
* 버그 수정 내역 관리

---

## 3. 생성 버전

### Version : v1.0.0

프로젝트의 첫 번째 완성 버전으로 설정하였다.

버전 정보

* v1.0.0 : 최초 완성 버전
* v1.1.0 : 기능 추가 버전
* v1.1.1 : 버그 수정 버전
* v2.0.0 : 대규모 기능 개선 버전

---

## 4. Release 생성 과정

프로젝트 최종 기능 구현 후 GitHub Release를 생성하였다.

1. 프로젝트 기능 구현 완료
2. 통합 테스트 수행
3. GitHub 저장소에 최종 코드 업로드
4. 버전 태그 생성
5. Release 생성
6. 최종 배포 진행

사용 명령어

```bash
git tag v1.0.0
git push origin v1.0.0
```

---

## 5. Release 포함 기능

v1.0.0 버전에는 다음 기능이 포함되었다.

* 시간표 등록
* 시간표 전체 조회
* 요일별 조회
* 시간표 수정
* 시간표 삭제
* 시간순 정렬
* 수업 알림
* 파일 저장
* 파일 불러오기

---

## 6. 최종 배포 결과

시간표 알림 프로그램의 모든 기능 구현과 테스트를 완료하였으며, v1.0.0 버전을 최종 Release 버전으로 설정하였다.

이를 통해 프로젝트의 버전을 체계적으로 관리할 수 있도록 구성하였다.
