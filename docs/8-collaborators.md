# 8. Collaborators

## 1. Collaborator 개요

Collaborator는 GitHub 저장소에 접근하여 프로젝트를 함께 관리할 수 있는 협업자를 의미한다.

본 프로젝트에서는 팀원 간 원활한 협업을 위해 GitHub Collaborator 기능을 활용하였다.

Collaborator를 등록함으로써 소스 코드와 프로젝트 문서를 공유하고 공동으로 프로젝트를 진행할 수 있었다.

---

## 2. Collaborator 등록 목적

시간표 알림 프로그램 프로젝트를 팀 단위로 진행하기 위해 Collaborator를 등록하였다.

등록 목적은 다음과 같다.

* 팀원 간 소스 코드 공유
* 프로젝트 문서 공동 작성
* GitHub Issues를 통한 업무 관리
* 프로젝트 진행 상황 공유
* 효율적인 협업 환경 구축

---

---

## 4. 협업 방식

프로젝트는 GitHub 저장소를 중심으로 협업을 진행하였다.

주요 협업 내용은 다음과 같다.

* 기능별 GitHub Issue 생성
* 프로젝트 문서 작성 및 관리
* 소스 코드 업로드 및 수정
* Commit 기록 관리
* 프로젝트 결과물 공유

---

## 5. Collaborator 등록 과정

1. GitHub 저장소 접속
2. Settings 메뉴 선택
3. Collaborators 메뉴 선택
4. 팀원 GitHub 계정 초대
5. 초대 수락
6. Collaborator 등록 완료

---

## 6. 권한 관리

Collaborator에게 프로젝트 수행에 필요한 권한을 부여하였다.

부여 권한

* 저장소 읽기
* 소스 코드 수정
* Commit 및 Push
* Issue 관리
* 프로젝트 문서 관리

협업 과정에서 필요한 범위 내에서 권한을 활용하였다.

---

## 7. 협업 결과

Collaborator 기능을 활용하여 팀원 간 프로젝트 자료를 효율적으로 공유할 수 있었다.

또한 GitHub를 통해 프로젝트 진행 상황을 관리하고 소스 코드를 공동으로 수정하며 최종 결과물을 완성할 수 있었다.

이를 통해 협업 능력과 GitHub 활용 능력을 향상시킬 수 있었다.
