# 📚 시간표 알림 프로그램

학생들이 개인 시간표를 쉽게 관리하고, 수업 시작 전에 알림을 받아 지각을 줄일 수 있도록 도와주는 Java 기반 시간표 관리 프로그램

---

# 📌 프로젝트 소개

시간표 알림 프로그램은 사용자가 수업 정보를 직접 등록하고 조회하며, 수업 시작 전에 알림을 받을 수 있도록 제작한 콘솔 기반 Java 애플리케이션입니다.

요일별 시간표 관리와 파일 저장 기능을 통해 효율적인 학습 일정 관리를 지원합니다.

---

# ✨ 주요 기능

## 📖 시간표 관리
- 수업 등록 / 조회 / 수정 / 삭제
- 요일별 시간표 조회
- 시간순 자동 정렬
- 강의실 정보 저장

## ⏰ 알림 기능
- 수업 시작 전 알림 시간 설정
- 콘솔 알림 메시지 출력

## 💾 파일 저장 기능
- 시간표 파일 저장
- 저장된 시간표 불러오기

## ⚠ 예외 처리
- 잘못된 입력 처리
- 중복 시간 검사
- 시간 형식 검증

## 2. 프로젝트 배경 및 범위

### 배경
대학생들은 요일마다 수업 시간이 다르고 강의실도 달라 혼동하는 경우가 많습니다. 또한 바쁜 일상 속에서 수업 시간을 놓치는 경우도 발생합니다. 이 프로젝트는 시간표를 등록하고 수업 시작 전에 알림을 제공하여 효율적인 학사 일정 관리를 돕기 위해 기획되었습니다.

### 포함 기능
- 수업 정보 등록
- 요일별 시간표 조회
- 수업 시간 수정 및 삭제
- 수업 시작 전 알림 기능
- 시간표 파일 저장 및 불러오기

### 제외 기능
- 모바일 앱 푸시 알림
- 학교 포털과 자동 연동
- 외부 데이터베이스 연동
- 위치 기반 알림 서비스

### 사용 기술
- Java
- IntelliJ IDEA
- GitHub

## 4. 요구사항 정의

### 4-1. 기능적 요구사항

| ID | 요구사항 | 상세 내용 | 우선순위 |
|---|---|---|---|
| F-01 | 시간표 등록 | 과목명, 요일, 시작 시간, 종료 시간, 강의실을 입력하여 시간표를 등록한다. | 높음 |
| F-02 | 시간표 전체 조회 | 등록된 모든 시간표를 목록 형태로 조회한다. | 높음 |
| F-03 | 요일별 조회 | 특정 요일을 입력하면 해당 요일의 수업만 조회한다. | 높음 |
| F-04 | 시간순 정렬 | 시간표 조회 시 요일과 시작 시간 기준으로 정렬한다. | 중간 |
| F-05 | 시간표 수정 | 등록된 수업의 과목명, 시간, 강의실 정보를 수정한다. | 높음 |
| F-06 | 시간표 삭제 | 사용자가 선택한 수업 정보를 삭제한다. | 높음 |
| F-07 | 수업 알림 설정 | 수업 시작 전 알림 시간을 설정한다. | 높음 |
| F-08 | 알림 메시지 출력 | 설정된 알림 시간에 콘솔에 알림 메시지를 출력한다. | 높음 |
| F-09 | 파일 저장 | 등록된 시간표 데이터를 파일로 저장한다. | 중간 |
| F-10 | 파일 불러오기 | 저장된 시간표 파일을 불러와 프로그램에서 다시 사용한다. | 중간 |
| F-11 | 샘플 데이터 추가 | 테스트를 위해 기본 수업 데이터를 추가할 수 있다. | 낮음 |
| F-12 | 잘못된 입력 처리 | 잘못된 메뉴 번호나 시간 형식 입력 시 오류 메시지를 출력한다. | 높음 |
| F-13 | 강의실 정보 등록 | 강의실 위치 정보를 입력하고 저장한다. | 중간 |
| F-14 | 중복 시간 검사 | 같은 시간대의 수업 등록 시 경고 메시지를 출력한다. | 중간 |
| F-15 | 종료 기능 | 사용자가 프로그램을 정상적으로 종료할 수 있다. | 높음 |

## 5. 시스템 설계

### 5-1. 전체 구조

사용자
↓
메인 메뉴(Main)
↓
시간표 관리(ScheduleManager)
↓
알림 기능(AlarmManager)
↓
파일 저장(FileManager)

---

### 5-2. 클래스 설계

| 클래스명 | 역할 |
|---|---|
| Main | 프로그램 실행 및 메뉴 출력 |
| Subject | 과목 정보 저장 |
| ScheduleManager | 등록, 조회, 수정, 삭제 기능 처리 |
| AlarmManager | 수업 알림 기능 처리 |
| FileManager | 파일 저장 및 불러오기 처리 |

---

### 5-3. 데이터 필드 설계

| 필드명 | 설명 |
|---|---|
| subjectName | 과목명 |
| day | 요일 |
| startTime | 시작 시간 (HH:mm) |
| endTime | 종료 시간 (HH:mm) |
| classroom | 강의실 |
| alarmTime | 알림 시간 (분 전) |

---

## 6. 추가 구현 노트

- 파일 저장 형식: JSON 권장 (가독성 및 확장성 우수). CSV도 옵션으로 제공 가능
- 시간 형식: HH:mm (24시간 표기)로 통일
- 중복 시간 검사: 동일 요일에 대해 시작~종료 시간이 겹치면 경고
- 알림 스케줄링: 프로그램 실행 중에만 동작 (추후 백그라운드 서비스로 확장 가능)
- 예외 처리: 입력 파싱 실패 시 친절한 오류 메시지 및 재입력 유도
