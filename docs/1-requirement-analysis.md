# Requirement Analysis

## 1. 프로젝트 개요

### 프로젝트명

시간표 알림 프로그램

### 프로젝트 목적

학생들이 자신의 시간표를 효율적으로 관리하고 수업 시작 전에 알림을 받아 지각을 예방할 수 있도록 지원하는 Java 기반 시간표 관리 프로그램을 개발한다.

---

## 2. 기능 요구사항

| 요구사항 번호 | 요구사항명      | 설명                                               |
| ------- | ---------- | ------------------------------------------------ |
| REQ-001 | 시간표 등록     | 과목명, 요일, 시작시간, 종료시간, 강의실 정보를 입력하여 시간표를 등록할 수 있다. |
| REQ-002 | 시간표 전체 조회  | 등록된 모든 시간표를 조회할 수 있다.                            |
| REQ-003 | 요일별 조회     | 특정 요일의 시간표만 조회할 수 있다.                            |
| REQ-004 | 시간표 수정     | 기존 시간표 정보를 수정할 수 있다.                             |
| REQ-005 | 시간표 삭제     | 선택한 시간표를 삭제할 수 있다.                               |
| REQ-006 | 시간순 정렬     | 등록된 시간표를 시작 시간 기준으로 정렬하여 출력할 수 있다.               |
| REQ-007 | 수업 시작 전 알림 | 수업 시작 전 알림 메시지를 출력할 수 있다.                        |
| REQ-008 | 파일 저장      | 현재 시간표 정보를 파일로 저장할 수 있다.                         |
| REQ-009 | 파일 불러오기    | 저장된 시간표 파일을 불러올 수 있다.                            |
| REQ-010 | 입력 예외 처리   | 잘못된 입력에 대해 예외를 처리하고 오류 메시지를 출력한다.                |
| REQ-011 | 중복 시간 검사   | 동일한 시간에 중복되는 수업 등록을 방지한다.                        |

---

## 3. 비기능 요구사항

### NFR-001 사용성

* 사용자는 메뉴 기반 인터페이스를 통해 쉽게 프로그램을 사용할 수 있어야 한다.
* 직관적인 메뉴 구성을 제공해야 한다.

### NFR-002 안정성

* 잘못된 입력이 발생해도 프로그램이 종료되지 않아야 한다.
* 예외 발생 시 적절한 오류 메시지를 제공해야 한다.

### NFR-003 유지보수성

* 기능별 클래스를 분리하여 구현한다.
* 코드의 재사용성과 가독성을 고려하여 설계한다.

### NFR-004 성능

* 시간표 조회 및 정렬 기능은 빠르게 수행되어야 한다.
* 저장 및 불러오기 기능은 데이터 손실 없이 수행되어야 한다.

---

## 4. 사용자 요구사항

### 사용자

* 대학생

### 사용자 요구

1. 시간표를 쉽게 등록하고 관리하고 싶다.
2. 요일별 수업 일정을 빠르게 확인하고 싶다.
3. 수업 시작 전에 알림을 받고 싶다.
4. 프로그램을 종료한 후에도 시간표를 저장하고 싶다.
5. 중복된 시간표 등록을 방지하고 싶다.

---

## 5. 제약사항

* Java 언어를 사용하여 구현한다.
* 콘솔 기반 프로그램으로 개발한다.
* JSON 파일을 사용하여 데이터를 저장한다.
* 외부 데이터베이스(DB)는 사용하지 않는다.
* Windows 환경에서 실행 가능해야 한다.

---

| 우선순위 | 요구사항               |
| ---- | ------------------ |
| 높음   | 시간표 등록, 조회, 수정, 삭제 |
