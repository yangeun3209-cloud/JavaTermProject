# Requirement Analysis

## 1. 요구사항 분석 개요

시간표 알림 프로그램을 구현하기 위해 필요한 기능적 요구사항과 비기능적 요구사항을 정리하였다.

---

## 2. 기능적 요구사항

| 번호 | 요구사항 | 설명 |
|---|---|---|
| 1 | 시간표 등록 | 과목명, 요일, 시작 시간, 종료 시간, 강의실 입력 |
| 2 | 전체 조회 | 등록된 모든 시간표 출력 |
| 3 | 요일별 조회 | 특정 요일의 수업만 조회 |
| 4 | 시간표 수정 | 등록된 시간표 정보 변경 |
| 5 | 시간표 삭제 | 선택한 시간표 삭제 |
| 6 | 시간순 정렬 | 요일과 시작 시간 기준 정렬 |
| 7 | 수업 알림 | 현재 요일과 시간 기준 수업 알림 |
| 8 | 파일 저장 | 시간표 정보를 파일로 저장 |
| 9 | 파일 불러오기 | 저장된 시간표 파일 불러오기 |

---

## 3. 비기능적 요구사항

| 항목 | 내용 |
|---|---|
| 사용성 | 콘솔 메뉴를 통해 쉽게 조작 가능 |
| 안정성 | 잘못된 입력에 대한 예외 처리 |
| 유지보수성 | 기능별 클래스를 분리하여 관리 |
| 확장성 | 추후 GUI 또는 DB 연동 가능 |
| 가독성 | 변수명과 메서드명을 명확하게 작성 |

---

## 4. 예외 처리 요구사항

- 숫자가 아닌 메뉴 입력 시 다시 입력하도록 처리
- 등록된 시간표가 없을 때 안내 메시지 출력
- 잘못된 수정 번호 입력 시 오류 안내
- 잘못된 삭제 번호 입력 시 오류 안내
- 파일이 없을 경우 불러오기 오류 안내
- 중복된 요일과 시간 입력 시 등록 차단

---

## 5. 요구사항 분석 결과

시간표 알림 프로그램은 기본적인 시간표 관리 기능과 파일 입출력 기능을 포함하도록 설계하였다.

또한 사용자가 잘못된 값을 입력하더라도 프로그램이 종료되지 않도록 예외 처리를 적용하였다.
