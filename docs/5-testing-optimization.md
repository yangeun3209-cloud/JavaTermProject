# 5. 통합 테스트 및 최적화

## 1. 통합 테스트

### 통합 테스트란?
통합 테스트는 개별 기능들이 함께 정상적으로 동작하는지 확인하는 과정이다.  
시간표 알림 프로그램에서는 시간표 등록, 조회, 수정, 삭제, 정렬, 알림, 파일 저장 및 불러오기 기능이 서로 문제없이 연결되는지 확인하였다.

---

## 2. 통합 테스트 항목 체크리스트

| 기능 | 정상 동작 | 예외 처리 | 비고 |
|---|---|---|---|
| 시간표 등록 | ✅ | ✅ | 빈 입력값 및 중복 시간 확인 |
| 전체 시간표 조회 | ✅ | ✅ | 등록된 시간표가 없을 경우 안내 |
| 요일별 조회 | ✅ | ✅ | 해당 요일 수업이 없을 경우 안내 |
| 시간표 수정 | ✅ | ✅ | 잘못된 번호 입력 처리 |
| 시간표 삭제 | ✅ | ✅ | 잘못된 번호 입력 처리 |
| 시간순 정렬 | ✅ | - | 요일 및 시작 시간 기준 정렬 |
| 수업 알림 | ✅ | - | 현재 요일과 시간 기준 알림 |
| 파일 저장 | ✅ | ✅ | timetable.dat 파일 저장 |
| 파일 불러오기 | ✅ | ✅ | 파일이 없을 경우 오류 안내 |

---

## 3. 테스트 방법

1. 프로그램 실행 후 시간표 등록 기능을 사용한다.
2. 등록된 시간표가 전체 조회에서 출력되는지 확인한다.
3. 특정 요일을 입력하여 요일별 조회가 정상 동작하는지 확인한다.
4. 등록된 시간표를 수정하고 변경 내용이 반영되는지 확인한다.
5. 등록된 시간표를 삭제하고 목록에서 제거되는지 확인한다.
6. 여러 시간표를 등록한 뒤 시간순 정렬이 되는지 확인한다.
7. 현재 요일과 시간을 입력하여 수업 알림이 출력되는지 확인한다.
8. 파일 저장 후 프로그램을 다시 실행하여 파일 불러오기가 되는지 확인한다.

---

## 4. 테스트 결과

전체 기능은 메뉴 선택 방식으로 정상 실행되었으며, 시간표 등록부터 조회, 수정, 삭제, 정렬, 알림 기능까지 정상적으로 동작하였다.  
또한 파일 저장 및 불러오기 기능을 통해 프로그램 종료 후에도 시간표 데이터를 다시 사용할 수 있음을 확인하였다.

---

## 5. 버그 추적 및 수정

### 발견된 문제
시간표 수정 또는 삭제 시 존재하지 않는 번호를 입력하면 오류가 발생할 수 있었다.

### 수정 방법
입력한 번호가 시간표 목록 범위 안에 있는지 검사하는 조건문을 추가하였다.

### 수정 결과
잘못된 번호를 입력하면 프로그램이 종료되지 않고 오류 안내 메시지를 출력하도록 수정하였다.

---

## 6. 코드 최적화

### 최적화 방향

1. 가독성 향상
   - 클래스별 역할을 분리하였다.
   - 변수명과 메서드명을 기능에 맞게 작성하였다.

2. 중복 코드 제거
   - 시간표 입력 기능을 `inputSubjectInfo()` 메서드로 분리하였다.

3. 예외 처리 강화
   - 숫자가 아닌 값을 입력했을 때 다시 입력하도록 처리하였다.
   - 파일 저장 및 불러오기 과정에서 발생할 수 있는 오류를 처리하였다.

4. 유지보수성 향상
   - `Subject`, `ScheduleManager`, `TimetableApp`, `Main` 클래스로 구조를 분리하였다.
