클래스 구조

시간표 알림 프로그램은 객체지향 설계를 적용하여 기능별 클래스로 분리하였다. 각 클래스는 하나의 역할만 담당하도록 설계하여 가독성과 유지보수성을 향상시켰다.

클래스 역할
Main 프로그램 실행 시작점
Subject 시간표 데이터 저장
ScheduleManager 시간표 관리 기능 처리
TimetableApp GUI 화면 및 사용자 입력 처리

Main 클래스

프로그램의 시작점으로 main 메서드를 포함한다. TimetableApp 객체를 생성하여 프로그램을 실행한다.

Subject 클래스

과목명, 요일, 시작 시간, 종료 시간, 강의실 정보를 저장하는 데이터 클래스이다. Serializable 인터페이스를 구현하여 파일 저장이 가능하도록 구성하였다.

ScheduleManager 클래스

시간표 등록, 조회, 수정, 삭제, 정렬, 파일 저장 및 불러오기 기능을 담당한다. ArrayList를 사용하여 시간표 데이터를 관리한다.

TimetableApp 클래스

사용자 인터페이스를 제공하는 클래스이다. 버튼 클릭 이벤트와 입력 처리를 담당하며 프로그램의 화면 구성을 관리한다.

설계 특징

  • 객체지향 설계 적용
  • 기능별 클래스 분리
  • 코드 재사용성 향상
  • 유지보수성 향상
  • 확장 가능한 구조 설계
