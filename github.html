GitHub 관리

본 프로젝트는 GitHub를 활용하여 소스 코드와 프로젝트 문서를 체계적으로 관리하였다.

GitHub를 통해 프로젝트 진행 상황을 기록하고 버전을 관리하며 팀원 간 협업을 수행하였다.

GitHub 활용 목적

  • 소스 코드 관리
  • 프로젝트 문서 관리
  • 버전 관리
  • 협업 환경 구축
  • 프로젝트 결과물 공유

프로젝트 저장소 구성

폴더 내용
src Java 소스 코드
docs 프로젝트 문서
README.md 프로젝트 소개
Release 버전 배포 관리

GitHub 사용 과정

  1. GitHub 저장소 생성
  2. 프로젝트 파일 업로드
  3. 문서 작성 및 수정
  4. Commit 생성
  5. Push 수행
  6. Release 생성
  7. GitHub Pages 배포

사용한 Git 명령어

git add .
git commit -m "프로젝트 구현"
git push origin main

git tag v1.0.0
git push origin v1.0.0

협업 내용

팀장 양은찬과 팀원 우민지는 GitHub를 활용하여 프로젝트 자료를 공유하고 문서를 작성하였다.

또한 Commit 기록을 통해 변경 내역을 관리하고 프로젝트 진행 상황을 확인하였다.

GitHub Pages 활용

GitHub Pages를 이용하여 프로젝트 발표용 웹페이지를 제작하였다.

프로젝트 개요, 기능 설명, 구현 내용, 테스트 결과 등을 웹페이지 형태로 정리하여 발표 자료로 활용하였다.

결과

GitHub를 통해 프로젝트를 체계적으로 관리할 수 있었으며 버전 관리와 협업 경험을 쌓을 수 있었다.
