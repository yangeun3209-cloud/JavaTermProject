# 시간표 알림 프로그램

## 프로젝트 목적
학생들이 개인 시간표를 손쉽게 관리하고, 수업 시작 전에 자동 알림을 받아 지각을 줄이며 학습 루틴을 개선하도록 돕는다.

## 주요 기능
- **시간표 등록**: 과목명, 요일, 시작/종료 시간 입력  
- **시간표 조회**: 주간 및 일별 보기 지원  
- **수업 시작 전 알림**: 사용자 지정 시간에 로컬 알림 전송  
- **수정 및 삭제**: 등록된 항목 편집·삭제 가능  
- **파일 저장/불러오기**: JSON 형식으로 로컬에 저장하고 복원 가능
---

# 1. 요구사항 정의

## 1-1. 기능적 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| F-01 | 시간표 등록 | 과목명, 요일, 시작시간, 종료시간 입력 후 저장 |
| F-02 | 시간표 조회 | 전체 시간표 및 요일별 시간표 조회 |
| F-03 | 시간표 수정 | 등록된 시간표 정보 수정 |
| F-04 | 시간표 삭제 | 등록된 시간표 삭제 |
| F-05 | 수업 알림 | 수업 시작 전 알림 제공 |
| F-06 | 파일 저장/불러오기 | 시간표 데이터를 파일로 저장 및 복원 |

## 1-2. 비기능적 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| NF-01 | 사용성 | 메뉴 기반으로 쉽게 사용 가능 |
| NF-02 | 안정성 | 잘못된 입력 시 오류 없이 재입력 가능 |
| NF-03 | 유지보수성 | 클래스 분리 구조로 수정 용이 |
| NF-04 | 성능 | 빠른 실행 및 응답 속도 제공 |

# 2. 시스템 설계

## 2-1. 전체 구조

사용자  
↓  
메인 메뉴(Main)  
↓  
시간표 관리(ScheduleManager)  
↓  
알림 기능(AlarmManager)  
↓  
파일 저장(FileManager)

## 2-2. 클래스 설계

| 클래스명 | 역할 |
|---|---|
| Main | 프로그램 실행 및 메뉴 출력 |
| Subject | 과목 정보 저장 |
| ScheduleManager | 등록, 조회, 수정, 삭제 |
| AlarmManager | 알림 기능 처리 |
| FileManager | 저장 및 불러오기 |

## 2-3. 데이터 필드

| 필드명 | 설명 |
|---|---|
| subjectName | 과목명 |
| day | 요일 |
| startTime | 시작 시간 |
| endTime | 종료 시간 |
| classroom | 강의실 |
| alarmTime | 알림 시간 |

## 2-4. 화면 설계

1. 시간표 등록
2. 시간표 조회
3. 시간표 수정
4. 시간표 삭제
5. 알림 설정
6. 저장 / 불러오기
0. 종료
