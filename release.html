Release

GitHub Release는 프로젝트의 특정 버전을 배포하고 관리하기 위한 기능이다.

본 프로젝트에서는 최종 완성 버전을 Release로 등록하여 프로젝트 결과물을 관리하였다.

Release 생성 목적

  • 프로젝트 버전 관리
  • 최종 결과물 배포
  • 기능 변경 내역 관리
  • 버그 수정 이력 관리

최종 버전 정보

버전 설명
v1.0.0 최초 완성 버전
v1.1.0 기능 추가 버전
v1.1.1 버그 수정 버전
v2.0.0 대규모 기능 개선 버전

v1.0.0 포함 기능

  • 시간표 등록
  • 시간표 전체 조회
  • 요일별 조회
  • 시간표 수정
  • 시간표 삭제
  • 시간순 정렬
  • 수업 알림
  • 파일 저장
  • 파일 불러오기
  • 예외 처리
  • 중복 시간 검사

Release 생성 과정

  1. 프로젝트 구현 완료
  2. 통합 테스트 수행
  3. GitHub 저장소 업로드
  4. Tag 생성
  5. Release 생성
  6. 최종 배포

사용 명령어

git tag v1.0.0
git push origin v1.0.0

결과

최종 버전인 v1.0.0 Release를 생성하여 프로젝트를 체계적으로 관리할 수 있도록 하였다.

이를 통해 프로젝트의 버전 관리와 배포 과정을 경험할 수 있었다.
