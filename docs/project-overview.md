# 시간표 알림 프로그램

## 프로젝트 목적
학생들이 개인 시간표를 손쉽게 관리하고, 수업 시작 전에 자동 알림을 받아 지각을 줄이며 학습 루틴을 개선하도록 돕는다.

---

# 1. 요구사항 정의

## 1-1. 기능적 요구사항

| ID | 요구사항 | 상세 내용 | 우선순위 |
|---|---|---|---|
| F-01 | 시간표 등록 | 과목명, 요일, 시작 시간, 종료 시간, 강의실을 입력하여 시간표를 등록한다. | 높음 |
| F-02 | 시간표 전체 조회 | 등록된 모든 시간표를 목록 형태로 조회한다. | 높음 |
| F-03 | 요일별 조회 | 특정 요일을 입력하면 해당 요일의 수업만 조회한다. | 높음 |
| F-04 | 시간순 정렬 | 시간표 조회 시 요일과 시작 시간 기준으로 정렬한다. | 중간 |
| F-05 | 시간표 수정 | 등록된 수업의 과목명, 시간, 강의실 정보를 수정한다. | 높음 |
| F-06 | 시간표 삭제 | 사용자가 선택한 수업 정보를 삭제한다. | 높음 |
| F-07 | 수업 알림 설정 | 수업 시작 전 알림 시간을 설정한다. | 높음 |
| F-08 | 알림 메시지 출력 | 설정된 알림 시간에 콘솔에 알림 메시지를 출력한다. | 높음 |
| F-09 | 파일 저장 | 등록된 시간표 데이터를 파일로 저장한다. | 중간 |
| F-10 | 파일 불러오기 | 저장된 시간표 파일을 불러와 프로그램에서 다시 사용한다. | 중간 |
| F-11 | 샘플 데이터 추가 | 테스트를 위해 기본 수업 데이터를 추가할 수 있다. | 낮음 |
| F-12 | 잘못된 입력 처리 | 잘못된 메뉴 번호나 시간 형식 입력 시 오류 메시지를 출력한다. | 높음 |
| F-13 | 강의실 정보 등록 | 강의실 위치 정보를 입력하고 저장한다. | 중간 |
| F-14 | 중복 시간 검사 | 같은 시간대의 수업 등록 시 경고 메시지를 출력한다. | 중간 |
| F-15 | 종료 기능 | 사용자가 프로그램을 정상적으로 종료할 수 있다. | 높음 |

---

## 1-2. 비기능적 요구사항

| ID | 요구사항 | 상세 내용 | 우선순위 |
|---|---|---|---|
| NF-01 | 사용성 | 사용자가 쉽게 이해할 수 있도록 메뉴 중심의 콘솔 UI를 제공한다. | 높음 |
| NF-02 | 안정성 | 잘못된 입력이 들어와도 프로그램이 종료되지 않고 다시 입력할 수 있어야 한다. | 높음 |
| NF-03 | 유지보수성 | 기능별 클래스를 분리하여 코드 수정과 기능 확장이 쉽도록 한다. | 중간 |
| NF-04 | 확장성 | 추후 GUI, 이메일 알림, 모바일 연동 기능을 추가할 수 있는 구조로 설계한다. | 중간 |
| NF-05 | 성능 | 프로그램 실행 후 빠르게 메뉴 화면이 출력되어야 한다. | 중간 |

# 2. 시스템 설계

## 2-1. 전체 구조

사용자  
↓  
메인 메뉴(Main)  
↓  
시간표 관리(ScheduleManager)  
↓  
알림 기능(AlarmManager)  
↓  
파일 저장(FileManager)

---

## 2-2. 클래스 설계

| 클래스명 | 역할 |
|---|---|
| Main | 프로그램 실행 및 메뉴 출력 |
| Subject | 과목 정보 저장 |
| ScheduleManager | 등록, 조회, 수정, 삭제 기능 처리 |
| AlarmManager | 수업 알림 기능 처리 |
| FileManager | 파일 저장 및 불러오기 처리 |

---

## 2-3. 데이터 필드 설계

| 필드명 | 설명 |
|---|---|
| subjectName | 과목명 |
| day | 요일 |
| startTime | 시작 시간 |
| endTime | 종료 시간 |
| classroom | 강의실 |
| alarmTime | 알림 시간 |

---
